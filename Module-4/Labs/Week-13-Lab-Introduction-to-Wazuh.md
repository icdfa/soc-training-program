# Week 13 Lab: Introduction to Wazuh

## Objective

To gain hands-on experience with Wazuh by installing it in your lab, deploying agents to your target machines, and analyzing the alerts.

## Scenario

You are a SOC analyst and your team has decided to deploy Wazuh as its new EDR solution. Your manager has asked you to set up a Wazuh server in your lab and deploy agents to your target machines.

## Instructions

### Part 1: Install the Wazuh Server

1. Follow the official Wazuh documentation to install the Wazuh server on a dedicated virtual machine (Ubuntu Server).
2. Access the Wazuh web interface and complete the initial setup.

### Part 2: Deploy Wazuh Agents

1. In the Wazuh web interface, go to **Agents > Deploy a new agent**.
2. Select the appropriate operating system for your target machines (Windows and Linux).
3. Follow the instructions to deploy the Wazuh agents to your target machines.
4. Verify that the agents are connected to the Wazuh server.

### Part 3: Analyze the Alerts

1. Generate some suspicious activity on your target machines, such as:
   - Running a vulnerability scan.
   - Creating a new user account.
   - Stopping a critical service.
2. In the Wazuh web interface, go to the **Security Events** dashboard.
3. Analyze the alerts generated by the suspicious activity.

## Deliverables

- Screenshots of the Wazuh dashboard showing the connected agents and the alerts generated by your suspicious activity.
- A brief write-up explaining the alerts and what they mean.
